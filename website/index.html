About CSV++

CSV++ (CSV Plus Plus) is an extension to the Comma-Separated Values (CSV) format defined in RFC 4180. It adds support for repeating fields (one-to-many relationships) and hierarchical component structures while maintaining backward compatibility with standard CSV parsers.

The Problem

Traditional CSV files excel at representing flat, tabular data. However, real-world data often contains:

  • Repeated values (e.g., multiple phone numbers for one person)
  • Structured components (e.g., addresses with street, city, state, zip)
  • Nested hierarchies (e.g., addresses with multiple address lines)

The Solution

While formats like JSON, XML, and YAML excel at representing hierarchical data, they introduce complexity and redundancy that may not be warranted for moderately structured datasets. CSV++ occupies a middle ground by extending CSV's tabular simplicity with hierarchical capabilities.

Design Principles

  1. Backward Compatibility: Standard CSV parsers can read CSV++ files
  2. Self-Documenting: Structure is defined in column headers
  3. Tabular Readability: Data maintains a tabular layout suitable for spreadsheets
  4. Explicit Over Implicit: Delimiters are declared, not assumed
  5. Recursively Composable: Structures can nest to any depth (though 3-4 levels recommended)

When to Use CSV++

CSV++ is most appropriate for:

  • Moderately structured data (1-3 levels of nesting)
  • Environments where CSV is already the established interchange format
  • Scenarios requiring backward compatibility with existing CSV infrastructure
  • Applications that benefit from self-documenting tabular data
  • Large datasets where file size and bandwidth matter

Note: For deeply nested hierarchical data (4+ levels), document-oriented formats like JSON or XML may provide better readability and tooling support.

Frequently Asked Questions

What is CSV++?

CSV++ is an extension to the standard CSV format (RFC 4180) that adds support for arrays and nested structures while maintaining full backward compatibility with existing CSV parsers.

Is CSV++ compatible with regular CSV?

Yes! CSV++ files are valid CSV files. Standard CSV parsers can open them, though they won't interpret the enhanced structure. The structure is encoded in the column headers using special syntax.

When should I use CSV++ instead of JSON or XML?

Use CSV++ when you need moderately structured data in a tabular format that's both human-readable and machine-parseable, especially when working with existing CSV-based systems or when file size matters.

How do I create arrays in CSV++?

Use square brackets in the header with a delimiter: phone[|] means the phone field contains multiple values separated by the pipe character. Empty brackets [] use the default delimiter (~).

How do I create structured fields?

Use parentheses to define components: address^(street^city^state^zip) creates an address structure with four components separated by the caret (^) character.

Can I combine arrays and structures?

Absolutely! You can have arrays of structures: address[~]^(street^city^state^zip) represents multiple addresses, each with structured components.

What delimiters should I use?

The specification recommends: ~ (tilde) for array values, ^ (caret) for structure components, ; (semicolon) for nested arrays, and : (colon) for deeply nested structures. You can use any character that doesn't conflict with your data.

How deep can I nest structures?

Technically unlimited, but we recommend keeping it to 3-4 levels for readability. Beyond that, consider using JSON or XML instead.

What file extension should I use?

Use .csv for maximum compatibility. You can optionally use .csvpp or .csvplus to explicitly indicate CSV++ format.

Are there any parsers or libraries available?

The specification is currently an IETF draft. We're working on reference implementations and welcome community contributions. Contact us if you're interested in contributing!
